什么是 Hexo？

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

安装前提

安装 Node.js

  • Node.js (Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)

    Node.js 版本限制

    强烈建议永远安装最新版本的 Hexo，以及 推荐的 Node.js 版本。

    Hexo 版本 最低兼容 Node.js 版本
    5.0+ 10.13.0
    4.1 - 4.2 8.10
    4.0 8.6
    3.3 - 3.9 6.9
    3.2 - 3.3 0.12
    3.0 - 3.1 0.10 or iojs
    0.0.1 - 2.8 0.10

4. 博客目录构成介绍

新建完成后，指定文件夹的目录如下：

  • node_modules
  • public
  • scaffolds
  • source
    • _posts
  • themes

_config.yml

网站的 配置 信息，您可以在此配置大部分的参数。

package.json

应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，可以自由移除。

node_modules

是node.js各种库的目录。

public

是生成的网页文件目录。

scaffolds

  • 模版文件夹。新建文章时，Hexo 会根据 scaffold 来建立文件。

  • Hexo的模板是指在新建的文章文件中默认填充的内容。

  • 里面就三个文件，存储着新文章和新页面的初始设置。

source

资源文件夹是存放用户资源的地方（包括文章、各类页面、图像等文件）。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题文件夹。Hexo 会根据主题来生成静态页面。
